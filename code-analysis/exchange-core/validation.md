### Путь: packages/exchange-core/src/utils/validation.ts

**Краткое назначение (1 предложение)**

Файл обратной совместимости, оставленный после рефакторинга валидационной системы на специализированные модули.

**Подробное описание (3–6 предложений)**

Модуль представляет собой placeholder файл, который был оставлен после крупного рефакторинга системы валидации для обеспечения обратной совместимости и предотвращения breaking changes в импортах. Ранее этот файл содержал все функции валидации, но они были перенесены в специализированные модули для лучшей организации кода и следования принципу единственной ответственности. Система теперь разделена на basic-validators для простых проверок, business-validators для доменной логики, composite-validators для сложных структур данных, и order-validators для полной валидации заявок. Центральный тип ValidationResult был перенесен в @repo/utils для централизованного использования во всех пакетах. Файл служит документацией архитектурного решения о реорганизации валидационной системы. Возможно, в будущем этот файл будет удален после полного перехода всех импортов на новые модули.

- Нет активных экспортов — файл содержит только комментарии
- Исторически экспортировал все валидационные функции до рефакторинга

**Риски и безопасность**

- **Breaking changes**: удаление файла может сломать legacy импорты
- **Migration completeness**: некоторые части кода могут все еще ссылаться на старую структуру
- **Documentation gap**: разработчики могут не знать о новой архитектуре валидации
- **Import confusion**: путаница в том, откуда импортировать валидационные функции
- **Orphaned references**: ESLint и другие конфигурации могут ссылаться на этот файл
- **Technical debt**: файл представляет технический долг до полной миграции
