### Путь: apps/admin-panel/app/error.tsx

**Краткое назначение (1 предложение)**

React Error Boundary компонент для обработки ошибок в административной панели с пользовательским интерфейсом восстановления.

**Подробное описание (3–6 предложений)**

Файл реализует специализированный error boundary для admin-panel приложения согласно Next.js App Router conventions. Компонент отображает пользовательский интерфейс ошибки с визуальным индикатором (иконка предупреждения), локализованными сообщениями на украинском языке и детализацией ошибки для debugging. Предоставляет пользователю два варианта восстановления: повторная попытка через функцию reset и навигация на главную страницу панели через прямую ссылку. Использует design system проекта с классами Tailwind CSS для консистентного стилинга и цветовой схемы error состояний. Компонент помечен как "use client" для работы в client-side контексте и обработки интерактивных событий.

**Экспортируемые сущности / API**

- `export default function Error` — React компонент для отображения ошибок
  - Props: `{ error: Error & { digest?: string }, reset: () => void }`
  - Возврат: JSX.Element с UI error страницы

**Входы (expected inputs) / Параметры**

- `error: Error & { digest?: string }` — объект ошибки с опциональным digest для трекинга
  - `error.message: string` — текст ошибки для отображения пользователю
  - `error.digest?: string` — уникальный идентификатор ошибки (Next.js специфично)
- `reset: () => void` — функция для повторной попытки рендеринга компонента

**Выходы / Побочные эффекты**

- Отображение error UI с деталями ошибки
- Навигация на `/admin` при клике на соответствующую кнопку
- Вызов функции reset для повторной попытки рендеринга
- Отсутствие логирования ошибок (должно быть добавлено)

**Взаимосвязи (кто вызывает / что вызывает)**

Файлы, которые используют этот компонент:

- Next.js App Router автоматически использует error.tsx при ошибках в `/admin` route
- Любые компоненты admin-panel при unhandled exceptions
- Server и client-side ошибки в административной панели

Файлы, которые импортируются здесь:

- Отсутствуют внешние импорты — использует только React и браузерные APIs

**Домен данных / типы**

```typescript
// Props интерфейс
interface ErrorProps {
  error: Error & { digest?: string };
  reset: () => void;
}

// Стандартный Error объект
interface Error {
  name: string;
  message: string;
  stack?: string;
}

// Next.js расширение с digest
interface NextJSError extends Error {
  digest?: string; // Уникальный идентификатор для серверных ошибок
}
```

**Риски и безопасность**

- **Information disclosure**: Показ error.message может раскрыть sensitive информацию
- **Client-side exposure**: "use client" делает компонент уязвимым для client-side атак
- **Error tracking**: Отсутствие логирования ошибок затрудняет debugging
- **User experience**: Технические сообщения могут быть непонятны пользователям
- **Navigation security**: Прямые ссылки могут обходить аутентификацию

**Тесты / рекомендации по покрытию**

- Unit тесты рендеринга с различными типами ошибок
- Integration тесты функциональности кнопок reset и навигации
- Visual regression тесты UI компонента
- Edge case тесты: ошибки без message, очень длинные сообщения
- Accessibility тесты для screen readers
- Error boundary тесты в реальных сценариях admin панели

**Оценка сложности (low/medium/high)**

**low** — простой UI компонент без сложной логики
