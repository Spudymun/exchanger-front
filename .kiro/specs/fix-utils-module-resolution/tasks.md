# План реализации: Исправление разрешения модуля @repo/utils

- [ ] 1. Добавить зависимость @repo/utils в веб-приложение
  - Добавить `"@repo/utils": "*"` в секцию dependencies файла `apps/web/package.json`
  - Выполнить `npm install` для обновления зависимостей
  - _Требования: 1.1, 4.1_

- [ ] 2. Настроить пути TypeScript для @repo/utils
  - Добавить пути `"@repo/utils": ["../../packages/utils/src"]` и `"@repo/utils/*": ["../../packages/utils/src/*"]` в `apps/web/tsconfig.json`
  - Убедиться, что пути следуют существующему паттерну для других пакетов
  - _Требования: 3.1, 3.2_

- [ ] 3. Проверить корректность экспортов в пакете utils
  - Убедиться, что `packages/utils/src/index.ts` экспортирует `input-validation` модуль
  - Проверить, что `packages/utils/src/input-validation.ts` содержит экспорт функции `useNumericInput`
  - Убедиться, что `packages/utils/package.json` имеет корректную конфигурацию exports
  - _Требования: 2.2, 2.3_

- [ ] 4. Протестировать разрешение модуля
  - Выполнить `npm run check-types --workspace=web` для проверки TypeScript
  - Запустить `npm run dev --workspace=web` для проверки работы в режиме разработки
  - Убедиться, что импорт `import { useNumericInput } from '@repo/utils'` работает без ошибок
  - _Требования: 1.2, 3.2, 3.4_

- [ ] 5. Протестировать функциональность useNumericInput
  - Открыть компонент `SendingCard.tsx` и убедиться, что импорт работает
  - Проверить, что функция `useNumericInput` возвращает `handleKeyDown` и `formatValue`
  - Протестировать работу валидации числового ввода в форме обмена
  - _Требования: 5.2, 5.3, 5.4_

- [ ] 6. Проверить автодополнение и типизацию в IDE
  - Убедиться, что IDE предлагает автодополнение для импортов из `@repo/utils`
  - Проверить, что Ctrl+Click переходит к определению функции `useNumericInput`
  - Убедиться, что TypeScript корректно типизирует возвращаемые значения функции
  - _Требования: 3.3, 2.4_

- [ ] 7. Выполнить полную сборку проекта
  - Запустить `npm run build --workspace=web` для проверки production сборки
  - Убедиться, что все зависимости корректно разрешаются в процессе сборки
  - Проверить, что не появились новые ошибки или предупреждения
  - _Требования: 1.3_

- [ ] 8. Протестировать интеграцию с формой обмена
  - Запустить веб-приложение и открыть форму обмена криптовалют
  - Протестировать поле ввода суммы с различными типами ввода (числа, буквы, специальные символы)
  - Убедиться, что валидация и форматирование работают корректно для разных валют
  - _Требования: 5.1, 5.4_
